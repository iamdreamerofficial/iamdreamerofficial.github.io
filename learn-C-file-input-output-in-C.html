File input/output (I/O) in C

Introduction to File I/O:

File I/O in C allows reading data from files and writing data to files. It provides a way to persist data beyond the lifetime of a program and interact with external files.

File Pointers:

A file pointer is a special variable used to keep track of the current position within a file. It is required for performing file I/O operations and is managed by the C library functions.

File Operations Modes:

There are different modes in which files can be opened in C:

  • "r" mode: Open a file for reading.
  • "w" mode: Open a file for writing. If the file already exists, its contents are truncated. If it doesn't exist, a new file is created.
  • "a" mode: Open a file for appending. Data is written at the end of the file. If the file doesn't exist, a new file is created.
  • "r+" mode: Open a file for both reading and writing.
  • "w+" mode: Open a file for reading and writing. If the file exists, its contents are truncated. If it doesn't exist, a new file is created.
  • "a+" mode: Open a file for reading and appending. Data can be both read and written, and new data is appended at the end of the file.

File Handling Functions:

C provides several functions for performing file I/O operations. Some commonly used functions include:

  • fopen(): Opens a file and returns a file pointer.
  • fclose(): Closes a file.
  • fgetc(): Reads a character from a file.
  • fputc(): Writes a character to a file.
  • fgets(): Reads a line from a file.
  • fputs(): Writes a string to a file.
  • fread(): Reads a block of data from a file.
  • fwrite(): Writes a block of data to a file.
  • fprintf(): Writes formatted data to a file.
  • fscanf(): Reads formatted data from a file.
  • feof(): Checks for the end-of-file indicator.

Opening and Closing Files:

Before performing any I/O operations, a file needs to be opened using the fopen() function. It takes two arguments: the file name (including the path) and the mode in which the file should be opened. Once the operations are complete, the file should be closed using the fclose() function to release system resources.

Reading from Files:

  • The fgetc() function reads a character from a file.
  • The fgets() function reads a line from a file and stores it in a string.
  • The fread() function reads a block of data from a file.
  • The fscanf() function reads formatted data from a file.

Writing to Files:

  • The fputc() function writes a character to a file.
  • The fputs() function writes a string to a file.
  • The fwrite() function writes a block of data to a file.
  • The fprintf() function writes formatted data to a file.

Error Handling:

File I/O operations can encounter errors, such as file not found, permission issues, or disk full. It is essential to handle these errors by checking the return values of file functions and using error handling techniques such as conditional statements or error codes.

Sample Code: Reading from a File

#include int main() { FILE* file = fopen("input.txt", "r"); if (file == NULL) { printf("File opening failed!"); return 1; } char c; while ((c = fgetc(file)) != EOF) { printf("%c", c); } fclose(file); return 0; }

The above code opens a file named "input.txt" in read mode using fopen(). It then reads characters from the file using fgetc() in a loop until the end-of-file indicator (EOF) is encountered. Each character is printed on the console. Finally, the file is closed using fclose().

Sample Code: Writing to a File

#include int main() { FILE* file = fopen("output.txt", "w"); if (file == NULL) { printf("File opening failed!"); return 1; } fprintf(file, "Hello, World!"); fclose(file); return 0; }

In this example, a file named "output.txt" is opened in write mode using fopen(). The fprintf() function is used to write the string "Hello, World!" to the file. Finally, the file is closed using fclose().

Error Handling:

It's important to handle errors during file operations. Checking the return values of file functions and using appropriate error handling techniques, such as printing error messages or returning error codes, ensures robust file I/O.

File I/O in C provides powerful capabilities for reading and writing data to external files. Understanding the concepts, modes, and functions involved allows you to manipulate files efficiently and handle potential errors effectively.
